# Testing

## Setup

Install pytest into the caiman conda environment (one-time):

```bash
pip install pytest
```

## Running unit tests

Unit tests require no data files and no GPU. Run them from the project root:

```bash
pytest tests/unit/```

These cover:
- `custom_df_f_startend` — z-score and norm-to-median normalisation math
- `get_resp1_resp2` — responder classification, threshold boundary, sort order, edge cases (zero responders, all respond)
- `_frame_layout` — frame count arithmetic including fractional frame periods
- `_remap_provenance_paths` — path remapping for Windows/Unix paths in nested provenance dicts

## Running regression tests

Regression tests verify that the analysis layer (`get_resp1_resp2`) still produces exactly the same outputs when given the same inputs as the ZH537 ground-truth run. They require two things on disk:

1. The ZH537 data folder (containing `analysis/` and `z1/`–`z5/` subdirectories)
2. Snapshot files generated by `update_snapshots.py` (see below)

Point the tests at your data folder by setting the `ZH537_DIR` environment variable before running:

```bash
# Windows
set ZH537_DIR=C:\path\to\your\ZH537

# or inline for a single command
ZH537_DIR="C:\path\to\your\ZH537" pytest tests/regression/```

If `ZH537_DIR` is not set, all regression tests are automatically skipped.

### Generate snapshots (run once)

```bash
set ZH537_DIR=C:\path\to\your\ZH537
python tests/regression/update_snapshots.py
```

This loads the ZH537 CNMF files, computes `stims1`/`stims2` arrays, and saves small snapshots (~300 KB) to `tests/regression/snapshots/`. Commit these snapshot files so the regression tests can run without the full ~5 GB dataset.

### Run regression tests

```bash
pytest tests/regression/```

Tests auto-skip on machines where the ZH537 data or snapshots are not present.

## After a CaImAn update (TensorFlow → PyTorch)

CaImAn CNMF outputs will change numerically — that is expected. The workflow to update the regression baseline:

1. Re-run the full pipeline on ZH537 with the new CaImAn version.
2. Visually verify the new heatmaps and neuron counts look reasonable.
3. Run `python tests/regression/update_snapshots.py` to regenerate snapshots from the new CNMF files.
4. Run `pytest tests/regression/ -v` — tests should pass (same analysis logic applied to new data).
5. Commit the updated snapshot files as the new ground truth.

The unit tests never need updating unless the analysis logic itself changes.
